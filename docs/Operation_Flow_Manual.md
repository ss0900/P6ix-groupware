# 영업관리 모듈 기능 플로우 매뉴얼

이 문서는 영업관리 모듈의 기능 흐름을 사용자 관점에서 설명한다. 모든 단계는 FE 화면 기준이며, 실제 업무 진행 순서에 맞춰 구성되어 있다.

## 1) 사전 준비

1. 로그인 가능한 계정이 준비되어 있어야 한다(영업담당/팀장/관리자).
2. 워크스페이스(회사) 권한이 설정되어 있어야 한다.
3. 프론트/백엔드가 정상 실행 중이어야 한다.

## 2) 권장 테스트 데이터(예시)

- 고객사: 세움건설
- 담당자: 홍길동 / 010-1111-2222 / hong@test.com
- 파이프라인: 민간공사
- 단계: 신규문의 → 현장미팅 → 견적제출 → 협상 → 수주(유형 수주) → 실주(유형 실주)
- 리드: 강남 A현장 리모델링 / 예상금액 50,000,000 / 유입경로 소개
- 견적 항목: 철거 1,000,000 / 인테리어 30,000,000
- 입찰: OO 공공청사 리모델링 입찰 / 보증금 1,000,000
- 매출 계획: 1차 기성 / 20,000,000
- 수금: 1차 기성 수금 / 20,000,000
- 연관계약: 계약 ID = 1001 (메모: 1차 계약)

## 3) 전체 흐름 개요

1. 파이프라인과 단계 설정
2. 고객사/담당자 등록
3. 리드(영업기회) 생성
4. 영업접수 처리
5. 파이프라인(칸반)에서 단계 이동
6. 리드 상세에서 활동/할 일/파일/견적/연관계약 관리
7. 견적서 생성 및 발송
8. TODO 캘린더 확인 및 완료 처리
9. 대시보드 KPI/전환/지연 리드 확인
10. 입찰 등록 및 캘린더 반영 확인
11. 매출 계획/수금 등록
12. 이메일 템플릿/서명/발송로그 관리

## 4) 화면별 상세 플로우

### 4-1. 파이프라인/단계 설정

1. 메뉴에서 `영업관리 > 파이프라인`으로 이동한다.
2. 상단에서 파이프라인 선택 드롭다운이 보이는지 확인한 뒤 `설정` 버튼을 클릭한다.
3. 좌측 파이프라인 영역에서 `추가` 버튼을 눌러 입력 폼을 연다.
4. 파이프라인 이름에 `민간공사`를 입력하고 `저장`을 클릭한다.
5. 목록에 `민간공사`가 표시되고 선택 상태로 전환되는지 확인한다.
6. 우측 단계 영역에서 `단계 추가` 버튼을 클릭한다.
7. 단계 입력 폼에서 단계명/유형/확률/색상을 입력하고 `저장`한다.
8. 예시로 아래 순서대로 등록한다(각 단계마다 `저장` 클릭).
9. `신규문의` / 유형 `진행중` / 확률 `10` / 색상 `파랑` 선택
10. `현장미팅` / 유형 `진행중` / 확률 `30` / 색상 `초록` 선택
11. `견적제출` / 유형 `진행중` / 확률 `50` / 색상 `노랑` 선택
12. `협상` / 유형 `진행중` / 확률 `70` / 색상 `주황` 선택
13. `수주` / 유형 `수주` / 확률 `100` / 색상 `파랑` 선택
14. `실주` / 유형 `실주` / 확률 `0` / 색상 `빨강` 선택
15. 단계 목록에 각 단계가 표시되고, 유형 라벨(진행중/수주/실주)과 확률(%)이 반영되는지 확인한다.
16. 좌측 상단의 뒤로가기 버튼을 눌러 파이프라인 보드로 돌아간다.
17. 파이프라인 보드에서 새 파이프라인이 선택 가능하며 단계 컬럼이 생성되는지 확인한다.

### 4-2. 고객사/담당자 등록

1. 메뉴에서 `영업관리 > 고객관리`로 이동한다.
2. 상단 우측의 `고객사 등록` 버튼을 클릭한다.
3. 모달에서 아래 정보를 입력한다.
4. 회사명: `세움건설`
5. 사업자번호: `123-45-67890`
6. 업종: `건설`
7. 연락처: `02-000-0000`
8. 주소: `서울시 강남구 테헤란로 123`
9. `저장` 버튼을 클릭해 등록한다.
10. 고객사 목록에 `세움건설`이 표시되는지 확인한다.
11. 목록에서 `세움건설` 행을 클릭해 상세 화면으로 이동한다.
12. 담당자 영역의 `추가` 버튼을 클릭한다.
13. 담당자 추가 모달에서 아래 정보를 입력한다.
14. 이름: `홍길동`
15. 직위: `과장`
16. 부서: `영업팀`
17. 이메일: `hong@test.com`
18. 전화번호: `02-111-2222`
19. 휴대전화: `010-1111-2222`
20. `주 담당자로 지정`을 체크한다.
21. `저장` 버튼을 클릭한다.
22. 담당자 목록에 `홍길동`이 표시되고 `주담당` 뱃지가 붙는지 확인한다.

### 4-3. 리드(영업기회) 생성

1. 메뉴에서 `영업관리 > 영업기회`로 이동한 뒤 `영업기회 등록` 버튼을 클릭한다.
2. 상단 헤더에서 화면 타이틀이 `새 영업기회`로 표시되는지 확인한다.
3. 기본 정보 영역에서 아래 값을 입력한다.
4. 제목(현장/프로젝트명): `강남 A현장 리모델링`
5. 파이프라인: `민간공사` 선택
6. 단계: `신규문의` 선택
7. 설명: `신규 문의 접수 후 현장 미팅 예정` 입력(선택)
8. 고객 정보 영역에서 아래 값을 입력한다.
9. 고객사: `세움건설` 선택
10. 담당자: `홍길동` 선택
11. 금액/일정 영역에서 아래 값을 입력한다.
12. 예상 금액: `50,000,000`
13. 예상 마감일: `2026-02-28`
14. 확률 (%): `10`
15. 다음 액션 예정일: `2026-02-10 10:00`
16. 유입 경로: `소개`
17. 하단의 `저장` 버튼을 클릭한다.
18. 저장 후 자동으로 목록 화면으로 이동하는지 확인한다.
19. 리스트에 리드가 생성되고, 단계/고객사/예상금액/담당자/다음 액션일이 표시되는지 확인한다.
20. 리스트 페이징을 이동해도 데이터가 정상 갱신되는지 확인한다.

### 4-4. 영업접수 처리

1. 메뉴에서 `영업관리 > 영업접수`로 이동한다.
2. 목록에서 리드를 하나 선택하고, 행 우측의 `접수 처리` 버튼을 클릭한다.
3. 모달 상단에서 리드 제목/고객사가 표시되는지 확인한다.
4. `담당자 지정`에서 담당자를 `홍길동`으로 선택한다.
5. `이동할 단계`에서 `현장미팅`을 선택한다.
6. `메모`에 `현장 미팅 일정 조율 필요`를 입력한다.
7. `다음 액션 TODO 생성`이 체크되어 있는지 확인한다.
8. TODO 제목에 `현장 미팅 일정 확정`을 입력한다.
9. 기한에 `2026-02-10 10:00`을 입력한다.
10. 담당자에서 `홍길동`을 선택한다.
11. `접수 완료` 버튼을 클릭한다.
12. 처리 후 해당 리드 상세 화면으로 이동되는지 확인한다.
13. 활동 로그에 `접수 처리`와 단계 변경 기록이 생성되는지 확인한다.
14. 할 일(또는 TODO) 목록에 새 TODO가 생성되는지 확인한다.
15. 리드의 다음 액션 예정일이 입력값으로 갱신되는지 확인한다.

### 4-5. 파이프라인(칸반)에서 단계 이동

1. 메뉴에서 `영업관리 > 파이프라인`으로 이동한다.
2. 상단 파이프라인 선택 드롭다운에서 `민간공사`를 선택한다.
3. `현장미팅` 컬럼에서 리드 카드(예: `강남 A현장 리모델링`)를 찾는다.
4. 해당 카드를 `견적제출` 컬럼으로 드래그 앤 드롭한다.
5. 카드가 `견적제출` 컬럼에 표시되고, 컬럼 상단의 건수 배지가 갱신되는지 확인한다.
6. 카드의 단계 색상/표시가 새로운 단계에 맞게 반영되는지 확인한다.
7. 카드를 클릭해 리드 상세 화면으로 이동한다.
8. 활동 로그에 단계 변경 기록이 생성되는지 확인한다.

### 4-6. 리드 상세 탭 활용

1. 리드 상세 화면 상단에서 리드 제목과 고객사가 표시되는지 확인한다.
2. `활동` 탭을 열고 우측 상단의 `활동 추가` 버튼을 클릭한다.
3. 활동 추가 모달에서 유형을 `전화`로 선택한다.
4. 제목에 `견적 일정 통화`를 입력하고, 내용에 `현장 방문 일정 공유`를 입력한다.
5. `저장` 버튼을 클릭한다.
6. 활동 목록에 새 항목이 표시되고, 최근 접촉일이 갱신되는지 확인한다.
7. `할 일` 탭으로 이동해 `할 일 추가` 버튼을 클릭한다.
8. 제목에 `견적서 초안 작성`을 입력한다.
9. 기한에 `2026-02-12 17:00`을 입력한다.
10. 담당자에 `홍길동`을 선택하고, 우선순위는 `보통`으로 둔다.
11. `저장` 버튼을 클릭한다.
12. 할 일 목록에 새 항목이 표시되고, 다음 액션 예정일이 갱신되는지 확인한다.
13. 생성된 할 일의 체크박스를 클릭해 완료 처리한다.
14. 완료 상태가 반영되고, 활동 로그에 완료 기록이 남는지 확인한다.
15. `파일` 탭으로 이동해 `파일 추가`를 클릭하고 임의 파일을 업로드한다.
16. 파일 목록에 업로드된 항목이 표시되는지 확인한다.
17. `견적` 탭으로 이동해 `견적 작성` 버튼을 클릭한다.
18. 견적서 작성 화면으로 이동되는지 확인한 뒤, 뒤로 이동해 리드 상세로 복귀한다.
19. `연관계약` 탭으로 이동해 `연관계약 추가` 버튼을 클릭한다.
20. 계약 ID에 `1001`, 메모에 `1차 계약`을 입력하고 `저장`한다.
21. 연관계약 목록에 항목이 표시되는지 확인한다.
22. 목록의 `삭제` 버튼을 클릭해 삭제가 가능한지 확인한다.

### 4-7. 견적서 생성 및 발송

1. `견적` 탭 또는 `견적서` 메뉴에서 `견적서 작성`을 클릭해 작성 화면으로 이동한다.
2. 기본 정보에서 `영업기회`를 `강남 A현장 리모델링`으로 선택한다.
3. 고객사/담당자가 자동으로 채워지는지 확인한다(수정 가능).
4. `견적 제목`에 `강남 A현장 리모델링 견적`을 입력한다.
5. `유효기간`을 `2026-03-15`로 입력한다.
6. `세율`은 `10`으로 둔다.
7. `견적 항목`에서 아래 항목을 입력한다.
8. 항목 1: 항목명 `철거`, 수량 `1`, 단가 `1000000`
9. 항목 2: 항목명 `인테리어`, 수량 `1`, 단가 `30000000`
10. 하단 합계(공급가액/세액/합계)가 자동 계산되는지 확인한다.
11. 필요 시 `머리말/꼬리말/거래조건/비고`를 입력한다(선택).
12. 하단의 `저장` 버튼을 클릭한다.
13. 저장 후 `견적서` 목록으로 이동하는지 확인한다.
14. 목록에 새 견적서가 생성되고 상태가 `작성중`으로 표시되는지 확인한다.
15. 해당 견적서 행을 클릭해 상세(수정) 화면으로 진입한다.
16. 기존 데이터가 로딩되고 항목 합계가 동일한지 확인한다.
17. 상단의 `PDF 미리보기` 버튼을 클릭해 PDF가 새 탭으로 열리는지 확인한다.
18. 상단의 `발송` 버튼을 클릭하고 확인창에서 확인한다.
19. 상태가 `발송됨`으로 변경되는지 확인한다.
20. 리드 상세의 활동 로그에 견적서 발송 기록이 생성되는지 확인한다.
21. `이메일` 화면의 `발송로그` 탭에서 발송 로그가 생성되는지 확인한다.

### 4-8. TODO 캘린더

1. 메뉴에서 `영업관리 > TODO 캘린더`로 이동한다.
2. 상단의 월 이동 버튼(좌/우)을 눌러 달력이 변경되는지 확인한다.
3. 캘린더 그리드에 일정이 표시되는지 확인한다.
4. 일정 유형이 리드 TODO, 리드 마감일, 견적 유효일, 입찰 마감일로 섞여 표시되는지 확인한다.
5. 오늘 날짜가 강조 표시되는지 확인한다.
6. 일정이 있는 날짜를 클릭한다.
7. 우측 패널에 해당 날짜 일정 목록이 표시되는지 확인한다.
8. TODO 항목에는 체크 아이콘과 `완료` 버튼이 함께 표시되는지 확인한다.
9. TODO 항목의 `완료` 버튼을 클릭한다.
10. 해당 TODO가 완료 상태로 변경되고 취소선이 적용되는지 확인한다.
11. 리드 상세로 이동해 다음 액션 예정일이 갱신되었는지 확인한다.

### 4-9. 대시보드

1. 메뉴에서 `영업관리 > 대시보드`로 이동한다(화면 타이틀: `영업 대시보드`).
2. 우측 상단에 `새로고침` 버튼이 표시되는지 확인한다.
3. KPI 카드가 4개 표시되는지 확인한다.
4. 카드 라벨이 `이번달 신규`, `진행중`, `수주`, `실주`인지 확인한다.
5. 각 카드에 숫자 값이 표시되는지 확인한다.
6. `단계별 전환` 섹션이 표시되는지 확인한다.
7. `예상 매출 합계` 값이 표시되는지 확인한다.
8. 파이프라인별 전환 테이블이 표시되고, 열 제목이 `단계/건수/전환율`인지 확인한다.
9. `지연 리드 TOP` 섹션이 표시되는지 확인한다.
10. 지연 리드 항목이 있을 경우 제목/지연일수/단계/예상금액이 표시되는지 확인한다.
11. 지연 리드 항목을 클릭했을 때 해당 리드 상세로 이동되는지 확인한다.

### 4-10. 입찰

1. 메뉴에서 `영업관리 > 입찰`로 이동한다.
2. 상단의 `입찰 등록` 버튼을 클릭한다.
3. 화면 타이틀이 `입찰 등록`인지 확인한다.
4. 입찰명에 `OO 공공청사 리모델링 입찰`을 입력한다.
5. 연관 리드에서 `강남 A현장 리모델링`을 선택한다(선택).
6. 상태는 `진행중`으로 둔다.
7. 마감일에 `2026-03-01 17:00`을 입력한다.
8. 보증금에 `1000000`을 입력한다.
9. 공고 URL에 `https://example.com/tender`를 입력한다(선택).
10. 설명에 `제출 서류 확인 필요`를 입력한다(선택).
11. 제출 서류에 `사업자등록증, 실적증명서`를 입력한다.
12. 하단의 `저장` 버튼을 클릭한다.
13. 저장 후 `입찰` 목록으로 이동하는지 확인한다.
14. 목록에 새 항목이 표시되고, 상태/마감일/보증금이 표시되는지 확인한다.
15. 캘린더에 입찰 마감 이벤트가 표시되는지 확인한다.

### 4-11. 매출/수금

1. 메뉴에서 `영업관리 > 매출/수금`으로 이동한다.
2. 상단 요약 카드(예정 합계/청구 합계/수금 완료/실수금/미수금)가 표시되는지 확인한다.
3. 매출 계획 영역의 `추가` 버튼을 클릭한다.
4. 매출 계획 모달에서 항목명에 `1차 기성`을 입력한다.
5. 연관 리드에서 `강남 A현장 리모델링`을 선택한다.
6. 계약 ID에 `1001`을 입력한다(선택).
7. 상태를 `계획`으로 둔다.
8. 예정금액에 `20000000`을 입력한다.
9. 예정일에 `2026-03-05`를 입력한다.
10. 비고에 `1차 기성 예정`을 입력한다(선택).
11. `저장` 버튼을 클릭한다.
12. 매출 계획 목록에 새 항목이 표시되는지 확인한다.
13. 수금 영역의 `추가` 버튼을 클릭한다.
14. 수금 모달에서 연관 리드에 `강남 A현장 리모델링`을 선택한다.
15. 매출 계획에 `1차 기성`을 선택한다.
16. 금액에 `20000000`을 입력한다.
17. 상태는 `예정`으로 둔다.
18. 수금 예정일에 `2026-03-10`을 입력한다.
19. 비고에 `1차 기성 수금`을 입력한다(선택).
20. `저장` 버튼을 클릭한다.
21. 수금 목록에 새 항목이 표시되는지 확인한다.

### 4-12. 이메일

1. 메뉴에서 `영업관리 > 이메일`로 이동한다.
2. 상단 탭이 `템플릿/서명/발송로그`로 표시되는지 확인한다.
3. `템플릿` 탭에서 `템플릿 추가` 버튼을 클릭한다.
4. 템플릿명에 `기본 견적 안내`를 입력한다.
5. 제목에 `견적서 안내드립니다`를 입력한다.
6. 본문(HTML)에 `<p>안녕하세요. 견적서 전달드립니다.</p>`를 입력한다.
7. 변수 스키마(JSON)에 아래 예시를 입력한다.
8. `{ "company_name": "string", "contact_name": "string" }`
9. `저장` 버튼을 클릭한다.
10. 템플릿 목록에 새 템플릿이 표시되는지 확인한다.
11. `서명` 탭으로 이동해 `서명 추가` 버튼을 클릭한다.
12. 서명명에 `영업팀 기본 서명`을 입력한다.
13. 서명 HTML에 `<p>홍길동 | 영업팀</p>`를 입력한다.
14. `기본 서명으로 설정`을 체크한다.
15. `저장` 버튼을 클릭한다.
16. 서명 목록에 새 항목이 표시되고 `기본` 표시가 붙는지 확인한다.
17. `발송로그` 탭으로 이동한다.
18. 로그 목록이 표시되는지 확인한다(없을 경우 빈 상태 메시지 확인).

## 5) 스크린샷 섹션(팀 공유용)

아래 항목은 문서 공유 시 필수로 첨부할 화면 캡처 목록이다. 화면 상단의 타이틀/버튼이 보이도록 캡처한다.

1. 영업관리 사이드바 전체(메뉴 라벨 확인)
2. `영업관리 > 파이프라인`(칸반) 상단 영역(파이프라인 선택 + `설정` 버튼)
3. 파이프라인 설정 화면(좌측 파이프라인 목록 + 우측 단계 목록)
4. `영업관리 > 영업기회` 목록 상단(`영업기회 등록` 버튼 포함)
5. `영업관리 > 영업접수` 목록 상단 및 `접수 처리` 모달
6. 리드 상세 화면(상단 헤더 + 탭 영역)
7. 리드 상세 `활동` 탭(활동 추가 모달 포함)
8. 리드 상세 `할 일` 탭(할 일 추가 모달 포함)
9. 리드 상세 `파일` 탭(파일 추가 버튼)
10. 리드 상세 `견적` 탭(견적 작성 버튼)
11. `견적서` 목록 상단 및 `견적서 작성` 버튼
12. 견적서 작성/수정 화면 상단(`PDF 미리보기`, `발송` 버튼 영역)
13. `TODO 캘린더`(월 캘린더 + 우측 일정 목록)
14. `대시보드` 화면 전체(상단 타이틀, KPI 카드, 지연 리드 TOP)
15. `입찰` 목록 상단 및 `입찰 등록` 버튼
16. 입찰 등록/수정 화면 상단
17. `매출/수금` 화면(요약 카드 + 매출 계획/수금 영역)
18. `이메일` 화면(템플릿/서명/발송로그 탭)

## 6) 권한/스코프 확인 포인트

- 영업담당: 본인 리드 수정 가능, 타인 리드 수정 불가
- 팀장: 팀(또는 그룹) 리드 수정 가능, 타인 리드 읽기 가능
- 관리자: 전 리드/견적/입찰/매출/수금 수정 가능
- 읽기 권한은 동일하고 수정/삭제 권한은 역할에 따라 제한된다
